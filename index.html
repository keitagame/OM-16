§ 03

CPU アーキテクチャ (OM-C1)

OM-C1 は 16-bit データバス・24-bit アドレスバスを持つ CISC プロセッサ。内部演算は 16-bit を基本とし、一部 32-bit 拡張演算をサポートする。パイプラインは 3 ステージ構成。

スプライト描画規則

⚠ ライン制限

1ラインに最大32枚のスプライトしか描画できない。33枚目以降は無視される。スプライトオーバーフローフラグ (PPUSTAT bit4) がセットされる。エミュレータは OAM 番号の小さい順に優先して描画すること。

規則詳細
優先度OAMインデックス 0 が最高優先 (手前に表示)。インデックスが大きいほど後ろ
透明色各パレットの色インデックス 0 は透明。背景が透けて見える
画面外X/Y が負またはX>256, Y>240 の場合は自動クリップ
フリップ大型スプライト(16×16以上)もピクセル単位でフリップ
DMAアップデートOAMはVBlank期間中にのみ安全に更新可能。DMA転送推奨
連結スプライト複数OAMエントリを隣接配置して大型キャラクタを表現可能

§ 24

エミュレータ実装上の注意事項

⚠ VRAM アクセス制限

PPUの描画中 (スキャンライン 0–239 のアクティブ期間) はVRAMへのCPU直接アクセスは禁止。VBlank期間 (スキャンライン 241–260) のみ安全。違反した場合はガーベジデータが描画されることがある。エミュレータは禁止期間のアクセスを無視するか警告を出すこと。

⚠ スプライト0ヒット

OAMインデックス0のスプライトの非透明ピクセルとBG0の非透明ピクセルが重なった最初のピクセルで PPUSTAT bit6 がセットされる。このフラグはスキャンライン0の開始時にクリアされる。ラスタースクロールの同期に使われることがある。

ℹ OAM優先度

同一スキャンライン上でスプライトが32枚を超える場合、OAMインデックス0–31が優先的に描画される（実機挙動）。エミュレータはOAMをスキャンし最初の32個の可視スプライトをピックアップすること。

ℹ DIV/MUL レジスタ副作用

MUL命令は R0 (下位16bit) と R1 (上位16bit) を上書きする。DIV命令は Rd (商) と R1 (余り) を書き換える。サブルーチン内でこれらを使用する場合は事前にレジスタを保存すること。
